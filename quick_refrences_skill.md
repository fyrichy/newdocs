# Script Storyboard Generator

剧本创作与分镜脚本提示词生成技能。本技能能够：
1. 接收剧本创作请求（.txt文件内容、标题+风格描述）
2. 发挥专业编剧才能进行剧本创作
3. 定义角色、场景、物品清单
4. 生成符合Qwen人物融合规范的分镜脚本提示词
5. 确保跨分镜的角色和场景高度一致性

---

## 工作流程

### 第一步：理解创作需求

用户会通过以下方式之一提供创作需求：

**方式A - 文件内容提供**：
```
以《d:\workspace\xxx.txt》内容，整体赛博朋克风格，然后发挥你好莱坞几十年专业编剧的才能，帮我进行剧本创作
```

**方式B - 标题+风格描述**：
```
以《星际穿越》标题，整体科幻史诗风格，然后发挥你好莱坞几十年专业编剧的才能，帮我进行剧本创作
```

**关键信息提取**：
- 原始素材来源（.txt文件或用户描述的概念）
- 目标风格（赛博朋克、科幻、文艺、悬疑等）
- 用户期望的基调（史诗感、紧张感、温馨感等）

---

### 第二步：剧本创作（好莱坞专业编剧标准）

使用三幕式结构进行剧本创作：

```
【剧本标题】
【副标题/Tagline】

【类型】类型标签
【时长】预估时长
【风格】详细风格描述

【一幕 - 建置】（约25%）
  【场景1】场景编号-场景描述
    - 时间：
    - 地点：
    - 主要人物：
    - 冲突引入：
    - 关键对白/动作：
    
【二幕 - 对抗】（约50%）
  【场景2】
  【场景3】
  【场景4】
  ...

【三幕 - 解决】（约25%）
  【场景N】
  【高潮】
  【结局】

【角色清单】
  角色1：姓名、年龄、外貌、性格、动机
  角色2：...

【场景清单】
  场景1：场景名称、视觉元素、光线、色调
  场景2：...

【物品/道具清单】
  - 关键道具1：外观、功能
  - 关键道具2：...
```

**剧本创作原则**：
1. **情感共鸣**：创造能引发观众情感共鸣的角色和情境
2. **视觉叙事**：每个场景都应有视觉亮点，适合图像化呈现
3. **节奏把控**：平衡紧张场景与舒缓场景
4. **冲突驱动**：每个场景都应有明确的冲突或悬念

### 第三步：分镜脚本提示词生成

参考`references/qwen_image_edit_character_fusion_guide.md`进行分镜脚本提示词编写。

#### 3.1 分镜脚本结构

```
【分镜1】
【景别】近景/中景/远景/全景
【角度】平视/俯视/仰视
【时长】X秒
【画面描述】
  主体：人物/物体描述
  背景：环境描述
  光线：光源方向、性质、强度
  色调：整体色调偏好
【人物状态】
  - 角色A：[动作]、[表情]、[服饰细节]
  - 角色B：[动作]、[表情]、[服饰细节]
【视角匹配】与前一分镜的透视关系
【风格匹配】与整体风格的统一性
【Qwen提示词】
[TASK_TYPE] 任务类型
[TARGET_SUBJECT] 目标主体描述
[LOCATION] 位置
[CHARACTERS] 角色详细描述
[ENVIRONMENT] 环境描述
[LIGHTING] 光线描述
[SHADOW] 阴影描述
[COLOR_GRADING] 色彩调校
[PERSPECTIVE] 透视关系
[TRANSITION_EDGE] 边缘过渡
[QUALITY] 输出质量要求
```

#### 3.2 Qwen提示词编写规范（来自人物融合指南）

**核心原则**：
1. **明确性原则**：描述越详细，结果越准确
2. **一致性原则**：保持人物特征在多帧/多图中的一致性
3. **上下文原则**：人物必须与场景环境相匹配
4. **细节原则**：描述光线、阴影、视角等技术细节

**人物描述模板**：
```
角色：姓名/身份
年龄：X岁
性别：X
种族：X
外貌：[面部特征]、[发型]、[肤色]、[身高]、[体型]
服装：[上衣]、[下装]、[鞋子]、[配饰]
姿态：[站姿/坐姿/动作]
表情：[面部表情]
```

**场景描述模板**：
```
场景：场景名称
地面/地板：[材质、颜色]
墙壁/背景：[材质、颜色、装饰]
家具/物品：[物品清单]
光线来源：[光源类型、方向、强度]
色调：[整体色调]
```

**融合规则**：
```
【任务类型】人物添加/人物替换/多人物场景融合
【替换区域】（如适用）使用掩码标记
【目标人物】详细人物描述
【环境融合】光线、阴影、透视、色调匹配
【过渡要求】边缘平滑过渡要求
【输出要求】质量标准
```

---

### 第四步：一致性保证

#### 4.1 角色一致性矩阵

创建角色一致性参考表，确保每个分镜中角色的外观、服饰、姿态保持一致：

```
【角色一致性参考】
角色名：
├─ 面部特征：双眼皮/单眼皮、高鼻梁、薄嘴唇等
├─ 发型：长度、颜色、造型
├─ 肤色：具体色号或描述
├─ 身高：具体数值或相对比例
├─ 体型：瘦/中等/健壮
├─ 服饰固定项：品牌、款式、颜色
├─ 配饰固定项：项链/耳环/手表等
└─ 表情习惯：微笑/严肃等
```

#### 4.2 场景一致性矩阵

创建场景一致性参考表：

```
【场景一致性参考】
场景名：
├─ 固定元素：不会改变的物体
├─ 可变元素：可能改变的物体
├─ 光线基调：主光源方向和色温
├─ 色调基准：整体偏暖/冷等
└─ 透视基准：视角、高度
```

#### 4.3 分镜衔接检查清单

在生成每个分镜后，检查：
- [ ] 角色服饰是否与上一分镜一致
- [ ] 角色位置关系是否合理
- [ ] 光线方向是否连贯
- [ ] 场景物体是否有逻辑延续
- [ ] 色调是否统一

---

### 第五步：输出完整交付物

最终输出包含：

1. **完整剧本文档**
   - 三幕式结构
   - 角色清单及详细设定
   - 场景清单及视觉描述

2. **分镜脚本提示词集**
   - 每个分镜的Qwen提示词
   - 角色一致性参考
   - 场景一致性参考

3. **技术规格说明**
   - 推荐分辨率
   - 风格参考标签
   - 可能的创意扩展建议

## 常见问题处理

### 问题1：角色外观漂移
**预防**：在角色清单中明确定义所有固定特征，并在每个分镜提示词中引用这些特征。

### 问题2：场景光线不一致
**预防**：创建场景光线基准，在每个分镜中标注与基准的关系。

### 问题3：分镜衔接跳脱
**预防**：每个分镜结尾描述下一分镜的开场状态，实现平滑过渡。
